# Sales Operations & Approvals — Handout

*Sales-Navigator · Great Plains Communications · Internal playbook*

**Paired references (dates on SharePoint):** Approval Overview & Training Guide; Approval Requirements Matrix; Change Order Process (Sales); Costing Routing Quick Reference; Move-Add-Change Processes; Salesforce Roadmap shared library. Use **Approval History** on the **GPC Solution**—not **OBR** materials—as the system of record for approvals. **OBR** = Operational Business Review readiness (forms/SOP)—account planning with ops leadership, **not** a substitute for Salesforce approval or costing rules.

## Approvals in Salesforce

- Path: **GPC Solution** → **Related List Quick Links** → **Approval History** → **Approve**, **Reject**, **Reassign**, or **Recall**; add comments; confirm; notifications go to assigned recipients.
- Leadership reviews **site-level** services, products, and **quote math**—not a single headline number.
- **Conga Files** → **Change Order** template → **Preview** from the GPC Solution when executing a narrow change-order motion.

## After signatures (change order package)

Upload signed PDF to the GPC Solution; complete package typically emailed to **Sales Manager** (when in workflow), **PMO**, **Salesforce Administrator**, **CRC**. **Salesforce Administration** performs booking adjustment; **PMO** updates project; **CRC** handles billing impacts.

**If** Solution Site project stage is **Pending Billing** or **7–10 Day Order**, Salesforce Admin must obtain **PMO/CRC approval** before executing the booking adjustment.

## Narrow “change order” vs MAC / disconnect / cancel

| Situation | Path |
|-----------|------|
| Contract change **after Closed/Won**, **before billing starts** | **Change order** (narrow definition)—preconditions include GPC Solution **Closed/Won**, **Booked**, **not Billed**; **cannot** run once **Billed** is set |
| **After billing**, partial in-life change | **Move-Add-Change (MAC)**—not the narrow change-order playbook |
| Remove everything **before** billing | **Cancellation** |
| Remove everything **after** billing | **Disconnect** |
| **Customer relocation** | **Customer Move** procedure (separate PDF in training library) |

Do **not** substitute **OBR** readiness for Salesforce approval, costing, or MAC/disconnect rules.

## Financial & investment language (Solution Site)

- **Regular Unit Price**, **Customer Unit Price**, discounts, totals.
- **Payback Months** ≈ Net Investment ÷ Gross Margin vs term.
- **Net Investment** (build, equipment, CAC-style costs—generally excluding labor).
- **Product Discount** outside standard tier / **EVPL** rules.
- **Churn** = percent drop from existing MRC to proposed MRC.
- **Unfavorable Investment:** **Gross Margin ≤ $0** and **Net Contract Value < $0**.
- **MCOS/NCOS** accuracy ties to correct **Salesforce Product** link for the Solution Site’s **Market Group**.

## Costing routing (high level)

- Routing follows **Site Type**, **Services**, **Products**, and Salesforce automation.
- **Request Manual Validation** on a Solution Site **skips** automation—each participating department must review.
- **New** + **Existing** on one site → routing follows **New**.
- **Multiple services** on one site → path requiring **deepest review**.
- **Zone Parent Opportunity:** Solution Sites may bypass automation; **Reusable** is **not** allowed when a Zone Parent is in play (see **Reusable** matrix PDF).
- **Wholesale** bulk imports follow a separate bulk-site routing pattern.
- **Strategic Build:** GPS inside preloaded boundaries → **Individual Drop Cost** and **Individual Equipment Cost** systematically.
- Edge cases: **Business Customer Care** for customer moves and New vs Existing classification.

## Change-order motion (summary)

1. Significant MRC / payback / cost → notify **Sales Manager** early.
2. Conga Change Order template → Preview.
3. Complete Solution Site, description, changes; new products → paste correct **Salesforce Product** for Market Group; **do not** delete **GPC Solution** number.
4. Collect signatures, upload PDF, email package + GPC Solution link.
